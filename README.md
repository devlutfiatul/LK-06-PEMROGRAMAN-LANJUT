# LK-06-PEMROGRAMAN-LANJUT

Secara keseluruhan, program pada repositori tersebut merupakan sistem perpustakaan sederhana yang menggunakan konsep OOP dan penyimpanan data berbasis file teks. Alur program dimulai dari method `main()` pada class `Main`, di mana pengguna pertama kali diminta melakukan login dengan memasukkan NIP. Sistem kemudian membaca file `pegawai.txt`, mengubah setiap baris menjadi objek `Pegawai` menggunakan method `fromFile()`, lalu mencocokkan NIP yang diinput dengan data yang ada. Jika cocok, pengguna berhasil login dan masuk ke menu utama; jika tidak, program langsung berhenti.

Setelah login berhasil, program masuk ke dalam perulangan menu (`while(true)`) yang memungkinkan pengguna memilih berbagai fitur, yaitu menambah siswa, melihat data siswa, melakukan peminjaman buku, pengembalian buku, dan melihat laporan. Pada fitur tambah siswa, data yang diinput pengguna akan dibuat menjadi objek `Siswa`, kemudian dikonversi ke format string (CSV) menggunakan method `toFile()` dan disimpan ke file `siswa.txt` melalui `FileHelper.write()`. Pada fitur lihat siswa, program membaca file `siswa.txt`, lalu setiap baris diubah kembali menjadi objek `Siswa` menggunakan `fromFile()` dan ditampilkan ke layar.

Pada proses peminjaman buku, sistem terlebih dahulu mengecek jumlah buku yang sedang dipinjam oleh seorang siswa melalui method `hitungPinjam()`, yaitu dengan membaca file `transaksi.txt` dan menghitung transaksi dengan status belum kembali (`status == 0`). Jika jumlahnya sudah mencapai batas (maksimal 2), maka peminjaman ditolak. Jika masih diperbolehkan, sistem akan membuat objek `Transaksi` baru dengan status 0 (belum kembali) dan menyimpannya ke file `transaksi.txt`.

Untuk proses pengembalian, program membaca seluruh isi file `transaksi.txt`, lalu mencari transaksi berdasarkan kode yang diinput pengguna. Jika ditemukan, status transaksi diubah menjadi 1 (sudah kembali). Setelah itu, seluruh data ditulis ulang ke file menggunakan `FileHelper.rewrite()`. Hal ini menunjukkan bahwa proses update data pada sistem ini dilakukan dengan cara membaca seluruh data, memodifikasi di memori, lalu menulis ulang ke file.

Pada fitur laporan, program kembali membaca file `transaksi.txt`, kemudian menampilkan hanya transaksi dengan status 0, yaitu buku yang belum dikembalikan. Secara umum, alur data dalam program ini selalu mengikuti pola yang sama, yaitu membaca data dari file, mengubahnya menjadi objek, memprosesnya sesuai kebutuhan, lalu (jika ada perubahan) menyimpannya kembali ke file. Dengan demikian, file teks berperan sebagai pengganti database, sementara class-class seperti `Siswa`, `Pegawai`, dan `Transaksi` berfungsi sebagai representasi data dalam bentuk objek.
